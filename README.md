# news-agent

Мониторинг новостей по тегам компаний: сбор из RSS открытых СМИ → сводка по важности через DeepSeek → красивое HTML-письмо с ссылками на первоисточники → отправка на email.

---

## Что делает

1. Читает теги из `tags.txt` (создаётся из `tags.txt.example`) и список RSS-лент из `feeds.txt`.
2. Скачивает новости из всех лент (ТАСС, РБК, Интерфакс, Коммерсантъ, Автостат, региональные и др.), фильтрует за последние сутки по тегам (по границе слова, без учёта регистра).
3. Отправляет найденное в DeepSeek — тот группирует события, ранжирует по важности и пишет сводку. Ссылки на первоисточники подставляются из исходных данных (модель их не выдумывает).
4. Верстает дайджест в HTML-письмо и доставляет на адрес из `MAIL_TO` (задаётся при установке).

## Быстрое развёртывание на новой машине (Ubuntu/Debian)

```bash
# 1. Скопировать проект на сервер (git clone или scp папки), затем:
cd news-agent

# 2. Запустить автоустановку (спросит ключ DeepSeek):
chmod +x deploy.sh
./deploy.sh

# либо без вопросов, передав ключ сразу:
DEEPSEEK_API_KEY=sk-ваш_ключ ./deploy.sh
```

Скрипт сам: ставит python3/venv/openssl, создаёт виртуальное окружение, ставит зависимости, делает `.env`, генерирует DKIM-ключ и **печатает DNS-запись**, которую нужно один раз опубликовать.

### После установки — 1 действие в DNS (для доставки во «Входящие»)

`deploy.sh` в конце выведет TXT-запись вида:

```
Имя:      newsagent._domainkey
Тип:      TXT
Значение: v=DKIM1; k=rsa; p=MIIBIjANBgkq...
```

Добавьте её в DNS домена отправителя (того, что указан в `DKIM_DOMAIN` / домен из email получателя; панель Yandex 360 → DNS, либо у регистратора). Проверить публикацию (подставьте свой домен):

```bash
./venv/bin/python -c "import dns.resolver; print(dns.resolver.resolve('newsagent._domainkey.ВАШ-ДОМЕН','TXT')[0].to_text()[:80])"
```

> Если переносите проект **вместе с папкой `keys/`** (через scp) — ключ переиспользуется, и менять DNS не нужно.

### Запуск

```bash
./venv/bin/python main.py            # полный цикл: сбор → сводка → отправка
./venv/bin/python main.py --dry-run  # без отправки, только сохранить HTML в out/
```

## Редактирование тегов и источников

- **Теги** — `tags.txt`, по одному на строку. Можно несколько вариантов написания одного бренда (Tesla / Тесла). Строки с `#` — комментарии. Свой `tags.txt` не коммитится; за основу возьмите `tags.txt.example` (`cp tags.txt.example tags.txt`).
- **Источники** — `feeds.txt`, формат `URL | Название`. Добавляйте/убирайте свободно — изменения применяются при следующем запуске.

Никакой перезагрузки или пересборки не нужно: оба файла читаются на старте `main.py`.

## Настройка (`.env`)

| Параметр | Назначение |
|----------|------------|
| `DEEPSEEK_API_KEY` | Ключ DeepSeek (обязательно) |
| `LOOKBACK_HOURS` | Глубина выборки в часах (по умолчанию 24) |
| `MAIL_TO` | Получатель письма |
| `SEND_MODE` | `direct` — свой мини-MTA (без аккаунта) / `relay` — через SMTP-аккаунт |
| `SMTP_FROM` | Адрес отправителя |
| `DKIM_SELECTOR`, `DKIM_DOMAIN` | Параметры DKIM-подписи |
| `SMTP_HOST/PORT/USER/PASSWORD/SSL` | Только для `SEND_MODE=relay` |

### Два режима доставки

- **`direct`** (по умолчанию) — письмо отдаётся напрямую на почтовый сервер получателя. Внешний SMTP-аккаунт не нужен, но требуется открытый исходящий порт 25 и опубликованная DKIM-запись.
- **`relay`** — отправка через авторизованный SMTP (например ящик на вашем домене, Yandex 360 → Пароли приложений). Самый надёжный способ попасть во «Входящие». Заполните `SMTP_*` и поставьте `SEND_MODE=relay`:
  ```
  SEND_MODE=relay
  SMTP_HOST=smtp.yandex.ru
  SMTP_PORT=465
  SMTP_USER=robot@ваш-домен.ru
  SMTP_PASSWORD=пароль_приложения
  SMTP_FROM=robot@ваш-домен.ru
  SMTP_SSL=true
  ```

## Автозапуск раз в день (опционально)

Через cron (например в 08:00):

```bash
crontab -e
# добавить строку (путь подставьте свой):
0 8 * * *  cd /home/USER/news-agent && ./venv/bin/python main.py >> out/cron.log 2>&1
```

## Структура проекта

```
deploy.sh          автоустановка на Ubuntu (одна команда)
main.py            точка входа
tags.txt           теги (НЕ коммитится; создаётся из tags.txt.example)
tags.txt.example   шаблон тегов
feeds.txt          RSS-ленты (редактируется)
.env               ключи и настройки (НЕ коммитится; создаётся из .env.example)
.env.example       шаблон настроек
keys/              DKIM-ключи (НЕ коммитятся)
requirements.txt   зависимости Python
src/config.py      загрузка настроек/тегов/лент
src/fetch.py       сбор RSS + фильтр по тегам и дате
src/summarize.py   сводка через DeepSeek
src/render.py      HTML-вёрстка письма
src/send.py        доставка (direct / relay) + DKIM-подпись
```

## Заметки

- Дзен и Яндекс.Новости не отдают публичный RSS, поэтому в источниках их нет; покрытие обеспечивают агрегирующие ленты СМИ, отраслевой Автостат и региональные издания.
- Если за сутки по тегам ничего нет — формируется письмо с пометкой «значимых новостей не найдено».
- Деливерабилити: при `direct` письмо подписывается DKIM; для гарантированных «Входящих» используйте `relay`.
